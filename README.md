# Secure Controller

Этот проект содержит простой Django-скрипт для работы с пропускной системой банка. Скрипт `main.py` подключается к базе данных и считает общее количество пропусков и количество активных пропусков.

## Что есть в проекте

- `main.py` — основной скрипт, который загружает настройки Django и выводит статистику по пропускам.
- `settings.py` — настройки проекта Django.
- `datacenter/models.py` — модели `Passcard` и `Visit`.
- `.env` — файл с переменными окружения для подключения к БД и секретным ключом.
- `requirements.txt` — зависимости проекта.

## Установка

1. Клонируйте репозиторий или скачайте файлы проекта.
2. Создайте виртуальное окружение и активируйте его.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Настройка окружения

Проект использует `python-dotenv`, чтобы загружать секреты и параметры базы данных из файла `.env`.

### Создание файла `.env`

1. Скопируйте файл `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```

2. Заполните файл `.env` реальными значениями для вашей базы данных.

### Переменные окружения

- **DB_HOST** — хост базы данных (адрес сервера PostgreSQL)
- **DB_PORT** — порт базы данных
- **DB_NAME** — имя базы данных
- **DB_USER** — имя пользователя для подключения к БД
- **DB_PASSWORD** — пароль пользователя базы данных
- **SECRET_KEY** — секретный ключ Django для шифрования. Должен быть уникальным и непредсказуемым

> **Важно:** Никогда не публикуйте файл `.env` в репозиторий. Он содержит конфиденциальные данные (пароли, токены). Используйте `.env.example` как шаблон для новых разработчиков.

## Запуск

Чтобы запустить скрипт:

```bash
python main.py
```

Скрипт подключится к базе данных, посчитает все пропуски и активные пропуски, а затем выведет результат.

## Примечания

- Для работы проекта требуется доступ к PostgreSQL и правильные значения из `.env`.
- В `settings.py` используются `os.getenv` и `load_dotenv()` для загрузки переменных окружения.
